# 前端修改说明：账号数据隔离

> **问题**：Tony 账号下 3 月 6 日记录的日记，在登录其他账号时也会出现。  
> **目标**：确保每个账号只看到自己的数据，切换账号后不显示上一账号的任何记录。

---

## 一、排查项

### 1.1 检查是否有硬编码 / Mock 数据

- 全局搜索：`3月6`、`3-6`、`2026-03-06`、`Tony`、`tony`
- 检查日记列表、首页高光、日历等页面是否混入了本地写死的示例数据
- **处理**：删除所有与真实用户数据混用的 mock / 假数据

### 1.2 检查数据缓存逻辑

- 日记列表、美德践行、愿景板等接口返回的数据是否有本地缓存
- 缓存 key 是否和用户 ID 或 Token 绑定（若未绑定，会导致跨账号串数据）

---

## 二、必须修改的逻辑

### 2.1 登出时清空用户相关数据

在用户登出时，务必清除：

| 数据类型 | 存储方式 | 操作 |
|----------|----------|------|
| Token | UserDefaults / Keychain / 内存 | 删除 |
| 用户信息 (user) | UserDefaults / 内存 | 删除 |
| 日记列表缓存 | UserDefaults / 内存 / 自定义缓存 | 清空 |
| 美德践行缓存 | 同上 | 清空 |
| 愿景板缓存 | 同上 | 清空 |
| 日历标记缓存 | 同上 | 清空 |
| 首页高光缓存 | 同上 | 清空 |

**示例（Swift）：**
```swift
func logout() {
    // 1. 清除 Token
    UserDefaults.standard.removeObject(forKey: "token")
    // 2. 清除用户信息
    UserDefaults.standard.removeObject(forKey: "user")
    // 3. 若有单独缓存 key，一并清除
    UserDefaults.standard.removeObject(forKey: "diaryListCache")
    // ...
}
```

### 2.2 切换账号 / 重新登录时

- 登录新账号后，**先清空上一账号的展示数据**，再请求新账号的接口
- 不要在「已有旧数据 + 新接口数据」上做追加，应先替换
- 若使用全局/单例的数据源（如 `@Published`、`ObservableObject`），切换账号时需重置为 `nil` 或空数组

### 2.3 缓存 key 需包含用户标识

若对接口数据做本地缓存以提升体验：

- 缓存 key 必须包含 `user_id` 或 `token`，例如：`diary_list_\(userId)` 或 `diary_list_\(token.prefix(20))`
- 否则不同账号会复用同一份缓存，导致数据串号

---

## 三、自测步骤

1. 用 Tony 账号登录，创建一条日记，确认能正常显示
2. 登出 Tony 账号
3. 用另一个账号登录
4. 进入日记列表 / 首页 / 日历
5. **预期**：不应再出现 Tony 的日记或 3 月 6 日相关记录
6. 在新账号下新建日记，确认只显示当前账号的数据

---

## 四、后端侧说明（供参考）

- 后端所有接口均按 `user_id`（来自 JWT）过滤，只返回当前登录用户的数据
- 3 月 6 日的异常记录已在数据库中删除
- 若前端按上述方式修改后仍有串号，可再排查网络层是否缓存了带 Token 的响应
